# Roadmap

This roadmap sequences the next product moves for UX RULER. `PRODUCT.md` remains the product memory: mission, audience, user need, assumptions, risks, metrics, and UX evaluation. This file exists because the product direction is now decided enough to choose what to do next.

Use this as a learning roadmap, not a feature backlog. Each item should stay tied to a problem, evidence, and signal.

## Direction

UX RULER should help humans and agents make better product decisions inside a repo before they build, by turning mission, audience, user need, evidence, value, metrics, and UX evaluation into compact durable context.

## Now

| Item | Problem Or Evidence | Signal |
|---|---|---|
| Make artifact handoffs explicit. | Recent landing-page measurement work added events and product-map updates, but the response ended as an implementation report instead of naming what to do next. | A future product-file edit closes with what changed, what decision it supports, what is measurable, what is missing, and 2 to 3 next options. |
